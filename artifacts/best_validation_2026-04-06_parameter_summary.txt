Policy-specific definitions:
- random:
  - budget ~ Uniform integer in [0, max_budget]
  - selected nodes sampled uniformly without replacement
- greedy:
  - one-hop immediate influence score per source node
  - score_i uses effective edge flow min(flow_t, dyn_cap_t) scaled by destination threshold
  - dynamic budget chosen by prefix objective: cumulative normalized score - reward_zeta * budget
  - reward_zeta (prefix cost weight): 0.05
- degree:
  - score_i = outdegree_i
  - same dynamic prefix budget rule as greedy
  - reward_zeta (prefix cost weight): 0.05
- lookahead:
  - candidate actions are greedy-ranked prefixes from budget 0..max_budget
  - chooses action with highest simulated discounted return over H steps
  - lookahead_h (used in latest strict baseline comparisons):
    - H=12 comparison: 4
    - H=24 comparison: 2
  - config default remains: cfg.greedy_lookahead_h = 6
  - lookahead_gamma: cfg.greedy_lookahead_gamma = 0.99
  - rollout assumes no additional interventions after first step for candidate evaluation
- storm:
  - uses RL policy with synthetic storm schedule (not a separate baseline policy head)
  - storm factor: 0.6
  - storm start: horizon // 2
  - storm duration: min(6, horizon // 3)
  - affected edges: top-capacity edges within configured incident edge fraction
